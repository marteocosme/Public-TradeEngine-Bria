---

# ✅ 🧠 Core Data Flow Pipeline
```
Market Data / Position State
↓
TradeEngine (Snapshot Builder)
↓
MM_SNAPSHOT_BEFORE
↓
Lifecycle Controller (Decision Only)
↓
Execution Layer (OrderSend / Modify)
↓
Result Integration
↓
MM_SNAPSHOT_AFTER
↓
Logger → File Output
```

---

# ✅ 📊 Snapshot Lifecycle

## ✅ BEFORE Snapshot

Generated by TradeEngine BEFORE any MM action.

### Contains:

- Current position state
- Market data (price, ATR)
- SL / TP / lot values
- mm_event_intent (initial)
- mm_phase

---

## ✅ AFTER Snapshot

Generated AFTER execution completes.

### Contains:

- Updated SL / TP / lot
- Exposure calculations
- ✅ Execution outcome
- Preserved event intent

---

# ✅ 🚀 Data Transformation Rules

## ✅ Rules

1. BEFORE snapshot is immutable baseline  
2. Execution modifies state  
3. AFTER snapshot reflects final state  
4. No data is lost between stages  

---

## ✅ Constraint

AFTER snapshot must always:

- Reflect actual execution outcome
- Retain original event intent
- Match schema column count

---

# ✅ 🔁 Data Consistency Guarantees

- BEFORE → AFTER must remain logically tied
- No recomputation outside TradeEngine
- No mutation inside Logger

---

# ✅ 🧩 Ownership Model

| Stage | Owner |
|------|------|
| Snapshot creation | TradeEngine |
| Decision making | Lifecycle Controller |
| Execution | Execution Layer |
| Logging | Logger |

---

# ✅ 🚫 Forbidden Operations

### ❌ Logger MUST NOT:
- Modify snapshot data
- Add/remove fields

### ❌ Lifecycle MUST NOT:
- Build snapshots
- Write logs

---

# ✅ 📌 Data Integrity Rules

- All snapshots must match Schema v1.2
- All rows must pass column validation
- Ordering must be preserved
