# Context Authority

This file defines how context conflicts are resolved.

When documents or interpretations disagree, the following order applies:

1. Project_Goals_and_NonGoals.md
2. Design_Decisions.md
3. Current_Project_State.md
4. Phase-X-Completion.md
5. Roadmaps
6. Architecture & Specs
7. Implementation

Rationale:
Intent and decisions must outlive implementation details.
